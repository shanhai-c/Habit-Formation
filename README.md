# Habit-Formation
一：关于项目
1.项目名称：Habit-Formation
2.项目概述：
本项目是一款基于HarmonyOS（鸿蒙系统）开发的轻量化习惯管理工具，采用ArkUI声明式UI框架与TypeScript语言构建，核心定位为“数据驱动的习惯养成助手”，帮助用户高效追踪习惯执行情况，提升自律性与习惯坚持的持续性。
核心目标
1. 为用户提供“新增-打卡-追踪-统计”全流程的习惯管理能力，覆盖健康、学习、工作、生活等多场景习惯；
2. 以数据为核心，通过今日完成率、连续坚持天数、分类完成率等关键指标，直观呈现习惯养成进度；
3. 打造简洁易用、视觉友好的交互体验，降低习惯管理的操作成本，提升用户使用意愿。
核心功能
习惯管理：支持自定义新增习惯（设置名称、分类、目标天数）、今日习惯打卡、分类筛选全部习惯、确认式删除习惯等核心操作；
数据统计：提供总习惯数、今日完成数、最长连续天数、月度完成率等核心数据概览，支持分类完成率统计；
基础交互：时段化问候语、日期/周数实时展示、未读通知标记、路由跳转（详情页/通知页/设置页）、操作Toast反馈等；
状态适配：空状态友好提示、习惯完成/未完成状态区分、分类专属配色体系，确保视觉一致性与用户体验连贯性。
适用于需要培养自律习惯的广泛用户群体（学生、职场人士、健身爱好者等），可用于追踪早起、阅读、运动、学习、写日记等各类长期习惯，帮助用户通过数据反馈感知进步，强化习惯养成的动力。
二：应用设计
整体功能可分为以下几大模块：
核心基础功能
1. 用户界面与基础交互
   顶部导航栏：展示用户信息（含时段化问候语）、通知入口（支持未读红点标记）、设置入口，点击可跳转对应页面；
   日期卡片：显示当前日期、周数，以及今日完成数、总习惯数、月度完成率等核心数据概览；
   标签页切换：支持“今日习惯/全部习惯/统计”三个核心页面的快速切换，配合视觉高亮和下划线标识当前选中态。

2. 习惯核心管理
   今日习惯：展示当日需完成的习惯列表，空状态下有友好提示；支持习惯打卡，打卡后更新连续天数、完成状态，并实时反馈打卡结果；
   全部习惯：支持按“健康/学习/工作/生活”分类筛选，展示所有习惯列表，空状态引导添加新习惯；
   习惯卡片：统一展示习惯名称、图标、提醒时间、坚持天数/目标天数、进度条，点击卡片可跳转详情页；
   新增习惯：通过悬浮按钮唤起弹窗，支持自定义习惯名称、选择分类、设置目标天数，提交后自动加入习惯列表；
   删除习惯：支持弹窗确认式删除，删除后同步更新统计数据。

 数据统计与可视化
1. 统计概览：以卡片网格形式展示总习惯数、今日完成数、最长连续天数、月度完成率，直观呈现核心数据；
2. 习惯坚持排行：按连续天数排序展示TOP5习惯，标注排名（冠亚季军特殊标识）、分类、目标天数、连续天数；
3. 分类统计：按“健康/学习/工作/生活”维度统计，展示各分类下习惯总数、完成数、完成率，配合进度条可视化呈现。

 辅助功能与细节优化
1. 时间与状态感知
   自动更新当前日期、小时，动态调整问候语（凌晨/早上/上午/中午/下午/傍晚/晚上）；
   实时计算周数、今日完成率、习惯进度（连续天数/目标天数）、分类完成率等数据；
2. 交互反馈：操作（打卡/新增/删除/输入为空）均有Toast提示，提升操作感知；
3. 数据模拟与更新：内置模拟习惯数据，页面展示时自动加载；打卡、新增、删除习惯后，实时更新统计数据；
4. 路由跳转：支持从首页跳转到通知页、设置页、习惯详情页，跳转时可传递习惯ID参数。

核心逻辑与规则
1. 打卡规则：当日未打卡时显示打卡按钮，打卡后标记为“已完成”，重复打卡有防重复提示；连续打卡时，若昨日已打卡则连续天数+1，否则重置为1；
2. 统计规则：
   今日完成率=今日完成习惯数/今日需完成习惯数×100%；
   分类完成率=分类内完成习惯数/分类内总习惯数×100%；
   月度完成率简化为当日完成率，最长连续天数取所有习惯的连续天数最大值；
3. 视觉规则：不同分类（健康/学习/工作/生活）对应专属配色，习惯卡片、统计模块均遵循该配色体系，提升视觉一致性。

整体而言，该软件围绕“习惯养成”核心，覆盖了习惯的新增、打卡、筛选、删除、详情查看，以及多维度数据统计分析，兼顾了功能完整性与交互友好性，核心目标是帮助用户追踪、管理习惯，提升习惯养成的持续性。
